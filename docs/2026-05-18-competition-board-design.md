# 竞赛信息板 - 设计方案

## 产品定位

混合型竞赛信息板，服务于咸鱼美术组社团：

- **大屏公告栏**：社团活动室循环展示近期竞赛
- **公开网页**：浏览器访问浏览赛事信息
- **微信分享**：分享链接到微信群，成员点击查看详情
- **管理后台**：管理员审核发布赛事，AI 辅助搜集

## 竞赛范围

国家级设计赛事、教育部认证比赛为主，包括但不限于：

- NCDA 未来设计师·全国高校数字艺术设计大赛
- 全国三维数字化创新设计大赛（3D大赛）
- 米兰设计周——中国高校设计学科师生优秀作品展
- 中国好创意暨全国数字艺术设计大赛
- GGAC 全球游戏美术概念大赛
- 白金创意国际大学生平面设计大赛
- 靳埭强设计奖 / KTK 设计奖
- 站酷百大设计榜 / 大设计奖

## 技术方案

### 整体架构

- **框架**：Next.js App Router（前后端一体）
- **数据库**：PostgreSQL（Neon 免费） + Drizzle ORM
- **AI**：Claude API / OpenAI API 做竞赛信息结构化提取
- **搜索**：Bing/百度搜索 API 做赛事发现
- **部署**：开发期 Vercel 免费 → 上线迁移国内服务器
- **认证**：单人密码登录（管理员就一个），不搞用户系统

### 开发工作流

前后端分离设计，API 契约先行：

1. 第 1 阶段：后端 API + 数据模型 + 管理后台 + AI 搜罗 + 基础前端
2. 第 2 阶段：Sttitch 出设计稿 → 替换前端页面组件
3. 迁移无忧：前后端共享 TypeScript 类型，设计稿只换 UI 不碰逻辑

## 数据模型

三层结构，适应不同类型的设计赛事：

### Layer 1：卡片层（固定字段，所有赛事必有）

| 字段 | 说明 |
|------|------|
| title | 赛事名称 |
| organizer | 主办方 |
| level | 级别（国家级 / 省级 / 行业） |
| overview | 一句话简介 |
| poster_url | 海报/封面图 |
| contest_type | 赛事类型（征稿 / 排行 / 展览） |
| tags | 标签（如：平面、插画、AIGC、CG、产品） |
| submission_deadline | 最近提交截止日期 |
| official_url | 官方链接 |
| status | draft / published / archived |
| source | manual / ai |

### Layer 2：结构详情（弹性字段，有就填）

| 字段 | 说明 | 格式 |
|------|------|------|
| deadlines | 多阶段时间节点 | [{label, date}, ...] |
| categories | 设计类别列表 | ["视觉传达", "动画视频", ...] |
| eligibility | 参赛资格 | 文本 |
| entry_fee | 报名费 | {amount, note} |
| awards | 奖项设置 | 文本 |
| spec_format | 作品格式要求 | 文本 |
| ai_policy | AIGC 政策 | 文本 |

### Layer 3：自由正文

| 字段 | 说明 |
|------|------|
| detail_body | 完整赛事通知/公告，Markdown 格式 |

## 页面结构（5 页）

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 竞赛卡片流，筛选/搜索 |
| 竞赛详情/分享 | `/c/[id]` | 完整详情，微信打开自动隐藏导航 |
| 管理登录 | `/admin` | 密码登录 |
| 管理列表 | `/admin`（登录后） | 草稿箱+已发布，含AI搜罗弹窗 |
| 编辑表单 | `/admin/edit/[id]` | 新建/编辑共用，id=new 时新建 |

## API 接口（6 个）

### 公开

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/competitions` | 竞赛列表（筛选/搜索/分页） |
| GET | `/api/competitions/[id]` | 竞赛详情 |

### 管理（需登录）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/admin/login` | 登录 |
| GET | `/api/admin/competitions` | 管理列表（含草稿） |
| POST | `/api/admin/competitions` | 新建赛事 |
| PUT | `/api/admin/competitions/[id]` | 编辑赛事 |
| POST | `/api/admin/ai-scout` | 触发 AI 搜罗 |

## AI 搜罗流程

1. 管理员点击"AI 搜罗"按钮
2. 后台搜索 API 关键词搜索 + 定点源检查
3. LLM 提取结构化赛事信息
4. 返回预览列表给管理员
5. 管理员勾选确认 → 入库草稿箱
6. 逐条审核 → 发布或删除

## 开发分期

### 第一期：核心可用

- 数据库建表 + 项目脚手架
- 管理后台（登录 + 手动添加/编辑/发布/下架）
- 公开首页（竞赛列表 + 筛选）
- 竞赛详情页（含微信分享优化）
- AI 搜罗功能（管理员手动触发）

### 第二期：设计迁移

- Sttitch 出设计稿
- 公开页面组件重写为设计稿风格
- 管理后台保持纯白功能风

### 第三期：搜罗自动化

- 定时自动搜罗（从手动触发升级为 Cron 定时）

### 冻结（不做）

- 成员报名/组队 → 需要用户系统、权限管理，复杂度翻 5 倍，偏离核心价值
- 浏览统计面板 → 对社团决策无实际指导意义
- 评论/留言 → 没人维护会变垃圾区
